# 内容营销服务获客

> 不装、不飘、持续在复利的普通人，带着方法论帮另一群普通人看到更大的可能性。

---

## 一、什么是「线索思维」

**核心概念**：每篇公众号文章不仅是独立内容，更是一条「线索」——牵引读者从单篇文章走向IP生态全貌。

线索思维的三个层次：

| 层次 | 含义 | 示例 |
|------|------|------|
| **内线索** | 文章内部的结构线索（开头→判断→分层→结尾） | "如果你是XX，那就YY"引导读者找到自己的位置 |
| **外线索** | 文章之间的关联线索（暗示后续话题、关联旧文） | 结尾"更深的打法我后面会展开" |
| **生态线索** | 文章到IP生态的入口线索（引导加微信、进社群、关注系列） | 结尾"这类思考我持续在写，欢迎链接" |

**写作目标**：读者读完后不只想"这篇不错"，而是想"这人还在写什么？我得关注他"。

---

## 二、核心定位（一句话）

**付能先生**：一线实战派，聚焦微信生态、用户增长、私域运营。写的是真实观察、真实判断、真实建议。不是评论家，不是培训师。

**四条价值观锚点**：
1. 复利是第一性 — 每篇文章回答"怎么做才能持续受益"
2. 站在普通人视角 — 大白话、分层建议、不假装精英
3. 祛魅不崇拜权威 — 不引用大佬语录，用自己的判断说话
4. 不干涉他人因果 — 给选项和判断依据，"没有好坏之分，只有选择"

> 详细人设档案（出身锚点、事业转折、方法论内核、反常识立场、目标人群）见 `references/ip-profile.md`

## 三、写作流程（五步法）

### Step 1：选题判断

用 **HKR 框架** 判断选题质量：

| 维度 | 自测问题 | 门槛 |
|------|---------|------|
| **H - 有痛点** | 读者看了会想"这说的就是我" | 必须通过 |
| **K - 有知识增量** | 能说出别人没说过或没说透的判断 | 必须通过 |
| **R - 有真实依据** | 基于真实案例、数据、观察或亲身经历 | 至少一项 |

**选题来源优先级**：
1. 自己正在做的业务或客户遇到的问题（最高优先级）
2. 微信生态、视频号、抖音、小红书等平台的新功能/规则变化
3. 行业里的反常现象（"大家都在说放弃，但我觉得还有机会"）
4. 个人成长、职场观察、读书思考（低频但高真诚度）

**不适合写的选题**：
- 只追热点但没有自己判断的
- 已被写烂、没有新角度的
- 自己没经历过、没观察过、也没访谈过的纯理论
- 会引发站队但讲不清底层逻辑的情绪型话题

### Step 2：确定原型

根据选题类型匹配文章原型：

| 读者问题类型 | 原型 | 参考文件 |
|-------------|------|---------|
| "平台出了新功能，怎么看？" | A — 平台变化解读 | `references/article-archetypes.md` |
| "几个工具选哪个？" | B — 工具/投放实操 | 同上 |
| "行业在发生XX变化，怎么办？" | C — 行业现象判断 | 同上 |
| "聊聊个人成长/价值观" | D — 个人成长/价值观 | 同上 |

> 每种原型的详细结构模板见 `references/article-archetypes.md`

### Step 3：写作

**通用结构模板**（适用于 80% 的文章）：

```
1. 开头：用一个具体场景/对话/问题/现象引入，3-5句内交代"我们在聊什么"
2. 背景/解释：大白话解释机制、规则或现状
3. 核心判断：用"在付少看来"或"我的判断是"引出
4. 分层论述：按不同人群/场景/阶段展开，每段一个分论点
5. 结尾：开放、共勉或邀请交流 + 线索暗示
```

**写作偏好**：场景开头 → 分点论述 → 数据支撑 → 行动收尾

> 详细风格规则（句式、口癖、情绪基调、开头/结尾技巧、绝对禁区）见 `references/style-guide.md`

### Step 4：线索叠加

每篇文章至少叠加一条线索：

| 线索类型 | 加法 | 示例 |
|---------|------|------|
| 内线索 | 结构本身引导读者找到自己的位置 | "如果你是初创老板，那就XX；如果你是成熟企业，那就YY" |
| 外线索 | 结尾暗示后续话题方向 | "更深的打法我后面会展开" |
| 生态线索 | 结尾引导进入IP生态 | "这类思考我持续在写，欢迎关注/链接" |

**线索叠加原则**：
- 不要承诺"下篇一定写XX"，而是暗示方向、引发期待
- 每篇至少1条外线索 + 1条生态线索
- 内线索自然嵌入分层建议中，不需要额外添加

### Step 5：四层质检

| 层级 | 检查内容 | 方法 |
|------|---------|------|
| **L1 硬规则** | 标题党词汇、宏大背景开头、综上所述结尾、绝对化恐吓、无解释术语 | 逐项排查 `references/style-guide.md` 绝对禁区 |
| **L2 风格一致性** | 是否像付少说话？短句够多？口癖是否自然？情绪是否冷静/真诚？ | 对比口癖表和参考文章 |
| **L3 内容质量** | 有真实案例/数据？核心判断有依据？分层建议可执行？无AI幻觉？读者能带走什么？ | HKR 重新检验 |
| **L4 活人感** | 是真人写的还是冰冷说明文？有"在付少看来"锚点？有真实经历/故事/情绪？结尾开放真诚？ | 自问"我会这样跟朋友说吗？" |

## 四、AI 协作边界

### AI 擅长做

- 补充行业背景、平台规则、竞品案例
- 把零散素材整理成结构草稿
- 提供类比、比喻、对比案例
- 检查文章逻辑自洽性
- 按风格规则重写段落

### 人必须自己做

- 核心判断和立场（AI 不能替你说"在付少看来"）
- 真实的故事、对话、踩坑经历
- 情绪的真诚表达
- 行业直觉和分层建议
- 最终定稿和发布决策

### 推荐工作流

```
人 → 提供选题 + 核心观点 + 真实素材 + 目标读者
AI → 按原型确定结构 + 补充背景信息
人 → 判断结构合理，调整核心论点
AI → 按风格规则扩写正文 + 叠加线索
人 → 四层质检审稿 → 修改定稿发布
```
